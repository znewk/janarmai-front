# JanarmAI — сценарии тестирования потребительской части (не админка)

Дев-сервер: `npm run dev` в папке `janarmai-front`, приложение на **http://localhost:5173**.
Админка (`/admin/*`) в этот документ не входит — там отдельная авторизация по логину/паролю.

Все данные — мок (без реального бэкенда), хранятся в `localStorage` браузера. Между сценариями
рекомендуется **сбрасывать демо-данные**, чтобы не путаться в накопленных тестовых карточках/транзакциях.

## Как сбросить демо-данные

1. Войти в любой аккаунт → вкладка **«Кабинет»** → внизу экрана кнопка **«Сбросить демо-данные»**.
2. Либо вручную: DevTools → Application → Local Storage → удалить все ключи `janarmai-*` → обновить страницу.

Сброс возвращает 5 стартовых пользователей, 2 компании, стартовые карты/историю — все новые
регистрации/транзакции, сделанные в ходе теста, удаляются.

## 1. Регистрация — 5 веток

Точка входа для всех веток: главный экран `/` (выбор канала, 2 кнопки):
- **«Через eGov / банковское приложение»** → `/register/egov-bvu`
- **«Приложение КМГ»** → `/register/kmg` → далее выбор типа лица (4 варианта)

### 1.1. eGov / БВУ — резидент ФЛ (самый короткий путь)

**Путь:** `/` → «Через eGov / банковское приложение»

| Шаг | Что видим | Что делать | Ожидаемый результат |
|---|---|---|---|
| 1 | Экран с уже подставленными ФИО/ИИН/телефон (сгенерированы случайно, каждый заход — новые) и галочкой «Данные подтверждены каналом» | Попробовать нажать «Продолжить» **не отмечая согласие** | Кнопка неактивна (disabled) |
| 2 | — | Отметить чекбокс «Я согласен(на) на обработку персональных данных…» | Кнопка «Продолжить» становится активной |
| 3 | — | Нажать «Продолжить» | Переход на авто-степпер проверки МВД (без ввода данных вручную) |
| 4 | Степпер «Проверка в базе МВД (ТС и страховка ОГПО)» | Дождаться результата (300–1500мс) | Успех почти всегда (ИИН генерируется «безопасным» — не оканчивается на 6/7/8/9) |
| 5 | Итоговая карточка: категория ТС + суточный лимит | Проверить: если ТС есть — «легковая, 100 л/сутки» или «грузовая, 300 л/сутки»; если своего ТС нет — «легковая, 100 л/сутки» (персональный лимит) | Нажать «Выпустить карту» |
| 6 | — | — | Переход сразу на `/card` **с баннером** «Карта успешно выпущена» + кнопка «Перейти в кабинет». Карточка MunaiCard с QR уже видна |

**Проверить отдельно:** т.к. ИИН/ФИО генерируются случайно при каждом заходе на экран — при повторном заходе (кнопка «Назад» и снова «Через eGov…») данные должны быть **другими**.

### 1.2. Приложение КМГ — ФЛ-резидент (полный путь, самый длинный)

**Путь:** `/` → «Приложение КМГ» → «ФЛ · резидент РК»

| Шаг | Экран | Действие / данные для проверки |
|---|---|---|
| 1 | Форма ФИО / ИИН / телефон (ручной ввод) | Заполнить: ФИО любое (мин. 3 симв.), ИИН — **12 цифр**, телефон — формат `+7XXXXXXXXXX` |
| 2 | Степпер «ГБД ФЛ» → «БМГ» | См. таблицу детерминированных ошибок ниже |
| 3 | Степпер «Проверка в базе МВД» | Тот же принцип по последней цифре ИИН |
| 4 | Итоговая карточка лимита | «Выпустить карту» → `/card` с баннером |

**Негативные сценарии (обязательно проверить оба):**
- ИИН, оканчивающийся на **`9`** → шаг ГБД ФЛ падает с ошибкой «не найден», показывается кнопка «Ввести данные заново» → она возвращает на форму ввода.
- ИИН, оканчивающийся на **`8`** → ГБД ФЛ ошибка «ФИО не совпадает».
- ИИН, оканчивающийся на **`7`** (и НЕ на 8/9) → ГБД ФЛ проходит, а шаг БМГ падает «телефон не привязан к ИИН».
- ИИН, оканчивающийся на **`6`** (и не 7/8/9) → ГБД ФЛ и БМГ проходят, а следующий шаг (проверка МВД: ТС+ОГПО) падает с ошибкой «Не удалось получить данные из базы МВД» + кнопка «Повторить проверку».

**Позитивные варианты (по последней цифре ИИН, только для шага МВД, если ИИН не 6/7/8/9):**
- `0` → своего ТС нет, но действует полис ОГПО → категория «легковая», лимит 100 л (персональный).
- чётная (`2`/`4`) → своё легковое ТС → 100 л.
- нечётная (`1`/`3`/`5`) → своё грузовое ТС → 300 л.

### 1.3. Приложение КМГ — ФЛ-иностранец

**Путь:** `/` → «Приложение КМГ» → «ФЛ · иностранец»

| Шаг | Экран | Действие |
|---|---|---|
| 1 | Номер паспорта + «Сканировать паспорт» + «Пройти liveness» | Ввести номер паспорта, нажать оба мок-шага по очереди (второй активен только после первого), затем «Продолжить» |
| 2 | Степпер «Беркут» | См. негативные сценарии ниже |
| 3 | Телефон (любой формат, без SMS) | Продолжить |
| 4 | Авто-степпер МВД (ТС/ОГПО) | Как в 1.2 |
| 5 | Итог: лимит **всегда «без лимита»**, «Льготный лимит не применяется — отпуск по предельной цене» | «Выпустить карту» → `/card` |

**Негативные сценарии (обязательно проверить оба):**
- Номер паспорта, содержащий **`DUP`** (в любом регистре, например `PDUP123`) → «Отказ в регистрации: дубликат паспорта» + кнопка «Попробовать снова» (возвращает на ввод паспорта).
- Номер паспорта, содержащий **`NF`** → «Отказ в регистрации: личность не подтверждена».
- Обычный номер (например `P1234567`) → проходит успешно.

**Важно:** у иностранца карта всегда без суточного лимита и без льготной цены — на карточке и на `/card` должно быть написано «без лимита» вместо числа, прогресс-бар лимита не показывается (только текущее потребление).

### 1.4. Приложение КМГ — ЮЛ-резидент (автопарк)

**Путь:** `/` → «Приложение КМГ» → «ЮЛ · резидент РК»

| Шаг | Экран | Действие |
|---|---|---|
| 1 | БИН компании + ФИО представителя + телефон, затем код ЭЦП (4 цифры) | БИН — 12 цифр (демо: `123456789012` — уже занят seed-компанией, для новой регистрации используйте любой другой 12-значный БИН, не оканчивающийся на 9). Код ЭЦП: **`1234`** |
| 2 | Степпер «ГБД ЮЛ» | БИН на `9` → ошибка «не найден» + «Ввести БИН заново» |
| 3 | Автопарк: форма ГРНЗ + категория (легковая/грузовая), кнопка «Добавить ТС» | Добавить минимум 1–2 ТС (например одно легковое, одно грузовое) — каждое появляется в списке с кнопкой удаления |
| 4 | Назначение водителей (опционально) на каждое ТС | Можно пропустить (не обязательно), можно ввести ФИО+ИИН и нажать «Проверить» — считается пройденным независимо от ИИН (шаг декоративный, не блокирует продолжение) |
| 5 | «Продолжить» | Переход на `/cabinet` с баннером «Карты успешно выпущены» — по одной карте на каждое ТС автопарка (100 л на легковую, 300 л на грузовую) |

**Негативный сценарий:** неверный код ЭЦП (не `1234`) → ошибка «Неверный код подтверждения», поле кода очищается для повторного ввода.

### 1.5. Приложение КМГ — ЮЛ-нерезидент

**Путь:** `/` → «Приложение КМГ» → «ЮЛ · нерезидент»

| Шаг | Экран | Действие |
|---|---|---|
| 1 | Ручной ввод: название компании, рег. номер, телефон (без проверки ГБД ЮЛ) | Любые значения |
| 2 | Паспорт+liveness директора, затем «Беркут» | Как в 1.3 — те же `DUP`/`NF` негативные сценарии |
| 3 | Автопарк (та же форма, что в 1.4) | Добавить 1+ ТС |
| 4 | Водители (опционально) | Как в 1.4 |
| 5 | «Продолжить» | `/cabinet` с баннером. **Все карты автопарка — без льготной цены** (нерезидент), но лимит по объёму (100/300 л) всё равно есть — в отличие от иностранца-ФЛ, у ЮЛ-нерезидента потолок остаётся, только без льготной цены |

## 2. Вход в уже созданный аккаунт (`/login`)

Экран `/` → «Уже есть аккаунт? Войти» → `/login`, три способа:

### 2.1. eGov Mobile / БВУ — мгновенный вход
«eGov Mobile / БВУ» → список ранее зарегистрированных аккаунтов этого канала (после сброса демо там будет один seed-пользователь **Асанов Ануар Бахытович**) → тап по имени → короткая надпись «Вход...» → автоматический переход на `/home`.

### 2.2. Приложение КМГ — телефон + SMS
«Телефон + SMS-код» → ввести номер:
- **`+77011234502`** (seed ФЛ-резидент, Ахметова Гульнара) — резидент
- **`+79261234567`** (seed иностранец, Smirnov Ivan) — нерезидент

→ «Продолжить» → экран SMS-кода, **демо-код `1234`** (можно проверить и негативный сценарий — любой другой код → «Неверный код подтверждения») → успешный ввод → `/home`.
Негативный сценарий на этом же шаге: номер, не зарегистрированный через КМГ (например случайный) → «Пользователь с таким номером не найден среди зарегистрированных через приложение КМГ».

### 2.3. ЮЛ — БИН/рег.номер + код
«БИН / рег. номер + код» → ввести:
- **`123456789012`** (резидент, ТОО «Жолжелдер Логистикс»)
- или **`RU-7743012345`** (нерезидент, ООО «ТрансЛогистик РУ»)

→ «Продолжить» → код подтверждения (ЭЦП), **демо-код `1234`** → `/home`. Неверный БИН → «Компания с таким БИН/рег. номером не найдена».

## 3. Карта, QR и демо-заправка (`/card`, доступна из нижнего таба «Карта»)

Это новая версия флоу — **кнопки «Симулировать заправку» больше нет**, заправка запускается тапом по QR.

| Шаг | Действие | Ожидаемый результат |
|---|---|---|
| 1 | Открыть вкладку «Карта» | Видна карточка MunaiCard (тёмно-синяя, с QR-кодом, остатком лимита), под ней — блок «Сброс лимита» / «Цена в пределах лимита», подпись «Нажмите на QR-код на карте, чтобы предъявить его на АЗС» |
| 2 | Если у пользователя несколько карт (например ФЛ-резидент КМГ с легковой+грузовой) — сверху появляется переключатель карт (пилюли) | Переключение между картами обновляет QR/остаток лимита |
| 3 | Тапнуть по QR-коду на карточке | Открывается **полноэкранная модалка** с крупным QR, под ней — пульсирующая зелёная точка «Ожидание сканирования · 30…29…с» |
| 4 | Подождать **~3 секунды**, ничего не нажимая | QR автоматически сменяется анимацией «сканирования» (бегущая полоса), текст «АЗС сканирует QR-код…» |
| 5 | Подождать ещё ~1.3с | Появляется степпер из 3 шагов: «Проверка лимита» → «Резерв объёма» → «Отпуск подтверждён АЗС» — шаги подсвечиваются последовательно |
| 6 | Дождаться завершения (~2с) | Экран «Заправка подтверждена» — чек с названием АЗС, топливом+объёмом, типом цены (льготная/предельная) и итоговой суммой; кнопка «Готово» |
| 7 | Нажать «Готово» | Модалка закрывается, на `/card` **остаток лимита уменьшился** на объём заправки |
| 8 | Перейти на вкладку «История» | Новая операция — **первая в списке** с теми же данными, что были в чеке |

**Проверить также:**
- Если закрыть модалку крестиком **до** завершения (на 1-й секунде ожидания или во время сканирования/степпера) — операция **отменяется**, транзакция не появляется в истории, остаток лимита не меняется.
- При достижении 80% суточного лимита — всплывает toast-уведомление «Достигнуто 80% суточного лимита» (можно спровоцировать несколькими заправками подряд).
- QR-код на карте периодически обновляется сам по себе (раз в 30с) — если постоять на экране подольше, видно, что код меняется.
- Тот же QR-таб-флоу работает и из личного кабинета ФЛ (см. п.4) — там у каждой карты свой тап по QR.

## 4. Личный кабинет ФЛ (`/cabinet`, вкладка «Кабинет» — показывается автоматически, если вход был как физлицо)

- Шапка: аватар-инициалы + ФИО, кнопка «Выйти».
- Список карт пользователя — у каждой карточки MunaiCard тап по QR тоже запускает полный сценарий из раздела 3 (с автозакрытием чека).
- Кнопка «Карта и QR-код» — переход на `/card`.
- «Последние заправки» — сгруппированный список (до 5 последних операций), ссылка «Вся история →» на `/cabinet/history`.
- Внизу — «Сбросить демо-данные» (см. начало документа) и «Выйти» (сверху).

**Проверить:** после «Выйти» — переброс на `/` (экран выбора канала), повторный заход на защищённые маршруты (`/home`, `/card`, `/cabinet`) без логина должен показывать «Нет активной сессии — сначала войдите или зарегистрируйтесь».

## 5. Личный кабинет ЮЛ (`/cabinet`, показывается для входа как компания)

- Шапка: аватар-инициалы + название компании, «Выйти».
- **Автопарк** — таблица ТС компании: ГРНЗ, категория, водитель (или «не назначен»), прогресс-бар лимита по каждой карте.
  - «+ Добавить ТС» — форма ГРНЗ+категория → «Добавить и выпустить карту» → сразу появляется в списке с новой картой.
  - Значок корзины у ТС → «деактивирует» ТС (убирает из списка активных).
  - «Назначить водителя» / «Снять водителя» на каждой строке.
- «История заправок» → `/cabinet/history` (те же операции, что и по всем картам автопарка, с фильтром по водителю дополнительно).
- «Скачать отчёт» — генерирует и скачивает CSV-файл со всеми операциями автопарка (проверить, что файл скачивается и открывается, колонки: дата/время, ГРНЗ, категория, водитель, АЗС, топливо, литры, тип цены, цена/л, сумма).
- Внизу — «Сбросить демо-данные».

## 6. Главная (`/home`, вкладка «Главная»)

- Шапка: аватар + приветствие («Здравствуйте, Имя!») + отображаемое имя/компания.
- Тёмная карточка сводки: остаток лимита на сегодня (сумма по всем картам пользователя/автопарка) + кнопка «Показать QR для заправки» (ведёт на `/card`).
- Сетка быстрых действий: «Карта / QR», «История», для ЮЛ дополнительно «Автопарк» и «Отчёт».
- «Последние заправки» (до 3 операций) со ссылкой «Все →».

## 7. История заправок (`/cabinet/history`, вкладка «История»)

- Фильтры сверху: период (весь/7 дней/30 дней), карта/ТС, АЗС, вид топлива, для ЮЛ — дополнительно водитель.
- Список — сгруппированная карточка с разделителями, отсортирован от новых к старым.
- Проверить: комбинации фильтров корректно сужают список; пустой результат показывает «Нет заправок по выбранным фильтрам».

## 8. Общие UX-моменты для проверки на любом экране

- Нижняя таб-навигация (Главная/Карта/История/Кабинет) — активная вкладка подсвечена, доступна на всех 4 защищённых экранах.
- Кнопка «Назад» в визардах регистрации (шеврон в кружке слева сверху) должна корректно возвращать на предыдущий шаг (а с первого шага ветки — на экран выбора типа лица/канала).
- На всех кнопках должен быть тактильный отклик при нажатии (лёгкое уменьшение/масштабирование).
- Отсутствие ошибок в консоли браузера (DevTools → Console) на любом из перечисленных экранов — если что-то красное появляется, это баг.

## Шпаргалка: детерминированные признаки ошибок (по последней цифре/подстроке)

| Проверка | Где встречается | Триггер | Результат |
|---|---|---|---|
| ГБД ФЛ | КМГ-регистрация ФЛ (1.2) | ИИН оканчивается на `9` | «не найден в ГБД ФЛ» |
| ГБД ФЛ | КМГ-регистрация ФЛ (1.2) | ИИН оканчивается на `8` | «ФИО не совпадает с ИИН» |
| БМГ | КМГ-регистрация ФЛ (1.2) | ИИН оканчивается на `7` | «телефон не привязан к ИИН» |
| МВД (ТС+ОГПО) | Все ветки ФЛ (1.1–1.3) | ИИН/паспорт оканчивается на `6` | «Не удалось получить данные из базы МВД» |
| МВД (ТС+ОГПО) | Все ветки ФЛ | оканчивается на `0` | нет своего ТС, есть ОГПО → 100 л |
| МВД (ТС+ОГПО) | Все ветки ФЛ | чётная (кроме 0/6) | своя легковая → 100 л |
| МВД (ТС+ОГПО) | Все ветки ФЛ | нечётная (кроме 6) | своя грузовая → 300 л |
